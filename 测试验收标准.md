# 测试验收标准

从 2026-02-28 起，本项目“通过测试验收”的定义固定为以下三条，三条必须同时满足，缺一不可。

## 1. 旧功能回归测试全部通过

- 任何功能迭代后，已有功能不得退化。
- 现有回归测试用于验证历史能力未被破坏。
- 当前执行入口：
  - 双击 `回归测试.command`
  - 或执行 `python -m unittest discover -s tests -v`

## 2. 新功能验收用例全部通过

- 每新增一个功能，必须新增对应的验收用例。
- 验收用例必须直接对应需求，而不是只验证内部实现。
- 验收用例至少应覆盖：
  - 正常流程
  - 关键边界条件
  - 失败或非法输入处理
- 未补齐新功能验收用例的改动，不视为完成。

## 3. 关键用户路径的系统测试全部通过

- 系统测试（ST / E2E）按真实用户操作路径执行。
- 目标是验证完整链路可用，而不是只验证局部函数。
- 关键路径至少包括：
  - 打开页面
  - 新增一笔记录
  - 删除一笔记录
  - 导出备份
  - 导入备份
  - 刷新后数据仍然正确显示
- 关键路径系统测试未通过，不允许验收通过。

## 执行规则

- 每次修改后，先跑自动化测试，再讨论交付。
- 任一已有测试失败，视为回归，必须先修复。
- 每修复一个 bug，必须补一个防复发测试。
- 没有测试覆盖的新功能，不允许标记为“已验收”。

后续所有功能迭代，均按本标准执行。
